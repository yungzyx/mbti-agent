# Profile index

Quick reference for the 16 MBTI profiles in `references/`.

| Type | Stack | Useful when the user wants | Main risk to control |
| --- | --- | --- | --- |
| INTJ | Ni-Te-Fi-Se | long-range architecture, strategy, decisive synthesis | over-compressing uncertainty |
| INTP | Ti-Ne-Si-Fe | causal debugging, model correctness, edge cases | analysis without closure |
| ENTJ | Te-Ni-Se-Fi | execution plans, prioritization, system-level decisions | moving too fast past human concerns |
| ENTP | Ne-Ti-Fe-Si | alternatives, design exploration, adversarial review | novelty without enough convergence |
| INFJ | Ni-Fe-Ti-Se | intent-aware design, principled review, synthesis | over-reading implicit goals |
| INFP | Fi-Ne-Si-Te | values-sensitive explanations, reflective product choices | avoiding direct tradeoff calls |
| ENFJ | Fe-Ni-Se-Ti | collaborative planning, team-facing explanation | consensus before technical precision |
| ENFP | Ne-Fi-Te-Si | ideation, motivation, user-centered options | too many possibilities |
| ISTJ | Si-Te-Fi-Ne | safe changes, precedent, regression control | rejecting useful novelty too early |
| ISFJ | Si-Fe-Ti-Ne | careful support, migration safety, user impact | over-preserving existing patterns |
| ESTJ | Te-Si-Ne-Fi | operational execution, checklists, delivery | bluntness or rigidity |
| ESFJ | Fe-Si-Ne-Ti | team/process fit, onboarding, support docs | prioritizing harmony over root cause |
| ISTP | Ti-Se-Ni-Fe | hands-on debugging, minimal mechanisms, runtime truth | under-explaining context |
| ISFP | Fi-Se-Ni-Te | humane UX, practical craft, careful changes | delaying hard structural calls |
| ESTP | Se-Ti-Fe-Ni | fast reproduction, incident response, tactical fixes | acting before enough safety checks |
| ESFP | Se-Fi-Te-Ni | demos, user-visible outcomes, momentum | local wins without long-term shape |
