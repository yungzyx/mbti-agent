# Cognitive functions

`mbti-agent` uses cognitive functions as a design rationale for coding workflow preferences. It does not claim that MBTI is scientifically definitive or that a type determines ability. The goal is user-controlled collaboration calibration.

## Function meanings for coding agents

| Function | Coding implication |
| --- | --- |
| Ni — Introverted Intuition | Converges on future-state patterns, architecture, hidden dependencies, and long-range consequences. |
| Ne — Extraverted Intuition | Generates alternatives, reframes problems, finds cross-domain possibilities, and resists premature closure. |
| Si — Introverted Sensing | Compares against precedent, known-good behavior, procedural memory, documentation, and stability. |
| Se — Extraverted Sensing | Engages immediate evidence, runtime behavior, concrete symptoms, and hands-on iteration. |
| Te — Extraverted Thinking | Organizes work by external results, efficiency, milestones, measurable outcomes, and execution order. |
| Ti — Introverted Thinking | Optimizes internal consistency, precise definitions, causal models, and logical correctness. |
| Fe — Extraverted Feeling | Optimizes shared understanding, team impact, collaboration, accessibility, and diplomatic feedback. |
| Fi — Introverted Feeling | Optimizes alignment with intent, values, user experience, authenticity, and humane tradeoffs. |

## Inferior-function stress

The inferior function is used only as a stress signal heuristic. Agents should not diagnose users. If a stress signal appears, reduce friction: simplify choices, clarify assumptions, and follow the relevant override note in the active type profile.
