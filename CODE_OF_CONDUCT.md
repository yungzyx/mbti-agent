# Code of conduct

This project is about coding-agent workflow adaptation, not judging people by personality labels.

Contributors are expected to:

- critique profile text and behavior descriptions, not people
- avoid type superiority, type insults, and stereotypes
- ground changes in observable coding workflow behavior
- respect that users may override or ignore MBTI entirely
- keep discussions practical, specific, and kind

Unhelpful contributions include:

- arguing that one type is better than another
- using MBTI as diagnosis or identity judgment
- adding roleplay or horoscope-like language
- dismissing user preferences because of a type label

Maintainers may close issues or pull requests that repeatedly violate these expectations.
